# Implementation Plan: Audio LLM Zero-Shot Baseline

**Branch**: `010-audio-llm-baseline` | **Date**: 2026-04-27 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/010-audio-llm-baseline/spec.md`

## Summary

Run Qwen2-Audio-7B-Instruct (a pure open-weights audio LLM) in zero-shot mode over the full 2183-clip seen-child test split to establish an "off-the-shelf frontier audio LLM" baseline. The model is prompted with "Is there a child vocalizing in this clip? Answer only: yes or no." Confidence is derived from the log-probability ratio of the "yes" token over the "no" token, threshold-tuned on the val set. Results are stored in `baselines/audio_llm_baseline_runs/qwen2_audio_7b/` matching the enrollment result folder schema, enabling a direct comparison row in the thesis baseline table.

## Technical Context

**Language/Version**: Python 3.11, `child-vocalizations` conda env
**Primary Dependencies**: `transformers>=4.45`, `accelerate`, `torchaudio`, `soundfile`, `pandas`, `scikit-learn`, `numpy`; optional: `bitsandbytes` for 4-bit quantization
**Storage**: Per-clip JSON cache at `baselines/audio_llm_cache/{model_slug}/`; result CSVs and JSONs at `baselines/audio_llm_baseline_runs/{model_slug}/`; no database
**Testing**: Smoke test via `--max-clips 10 --dry-run`; acceptance via predictions CSV schema check and metrics JSON field check
**Target Platform**: SLURM cluster (A100 40GB GPU node); `ou_bcs_normal` / `pi_satra` partitions
**Project Type**: CLI inference script + SLURM job script
**Performance Goals**: ≤ 12 hours walltime for 1309 test clips on a single A100; resume-safe (interrupted runs restart from cache)
**Constraints**: No fine-tuning or gradient updates; val-only threshold tuning; bfloat16 full precision (~18GB) preferred over quantization for cleaner logit extraction
**Scale/Scope**: 2183 clips total (437 val + 1309 test + 437 train), single model, two split runs
