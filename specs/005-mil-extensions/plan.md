# Implementation Plan: MIL Extensions

**Branch**: `005-mil-extensions` | **Date**: 2026-04-24 | **Spec**: [spec.md](spec.md)

## Summary

Extend the segment-instance MIL pipeline (feature 004) with three new aggregators (noisy-OR, top-k, transformer MIL), age-band stratified evaluation across 14-month and 36-month subsets, and a standalone weak-diarization evaluation script that measures how well MIL attention weights correlate with ground-truth child speech from existing RTTM files. The existing 16-config baseline results are preserved; new configs are added via resume-safe sweep extension. No new data, no new embedding computation.

## Implementation Design (per User Story)

### US1: Gated Attention Already Exists

`GatedAttnAgg` is already implemented in `mil/seg_model.py` (from feature 004). It wraps `GatedABMILHead` from `mil/mil_model.py`. Only change needed: ensure it is listed in `seg_mil_sweep.yaml`'s aggregators. **No new code for US1.**

### US2: Age-Band Stratification in seg_train.py

After test-set inference in `train_one_config()`, load `whisper-modeling/seen_child_splits/test.csv`, join on audio path to get `timepoint` column (`14_month` / `36_month`), and compute per-band metrics using the existing `compute_metrics()` + `tune_threshold()` helpers (threshold already tuned on val set — reuse it for each band). Write results to `test_metrics_by_timepoint.csv`. Add `test_auroc_14month` and `test_auroc_36month` to the `all_configs.json` entry.

### US3: Three New Aggregators in seg_model.py

**NoisyORAgg**: Per-instance linear head → sigmoid → log-space noisy-OR product → scalar bag logit. Mask applies neutral probability (1.0) to padding positions. Returns `(logit, None)`.

**TopKAgg**: Per-instance score head → mask padding with -inf → topk(k_actual) → mean-pool selected embeddings → final linear head. `k_actual = min(k_config, n_valid)`. Returns `(logit, None)`.

**TransformerAgg**: Prepend learned CLS token → sort bag by segment start time → add learned positional embeddings → 2-layer transformer encoder (4 heads, FFN=1536, dropout=0.3, pre-norm) → CLS output → linear classifier. Return CLS cross-attention weights scattered to K_max positions for interpretability. Returns `(logit, weights[K_max])`.

All three registered in `build_aggregator()`. Transformer HPs configurable in sweep YAML under `transformer_config:` key.

### US4: eval_weak_diarization.py

Standalone script. Inputs: `--results-dir`, `--split-csv`, `--rttm-cache`, `--output`.

Algorithm:
1. Discover all `test_segment_weights.csv` files under `results-dir/`.
2. For each file, load segment rows (audio_path, seg_start, seg_end, attention_weight).
3. For each segment, find RTTM file via `{stem}__{md5(audio_path)}.rttm` naming.
4. Parse RTTM, extract child-speaker lines (label contains `CHI`, `KCHI`, or `CHILD`).
5. Compute overlap between each segment and child-speaker segments → GT child fraction.
6. Join with split CSV on audio path → get `timepoint` per clip.
7. Compute Pearson, Spearman, AUROC (GT fraction ≥ 0.5 as binary label) per (frontend, aggregator, timepoint).
8. Write `weak_diarization_eval.csv`.

Skip configs where `test_segment_weights.csv` is absent (non-attention aggregators).
