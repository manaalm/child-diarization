## Summary

Build an audio-visual fusion pipeline for binary clip-level target-child vocalization detection as an experimental extension of the existing audio-only pipeline. The system extracts visual features from SAILS BIDS video clips (face detection/tracking + manual BIDS annotation fields already present in the split CSV), fuses them with audio baseline scores using XGBoost/logistic regression, and evaluates four model classes (audio-only, video-only, always-fuse AV, gated AV) with stratified reporting by age band and visual eligibility. A null or conditional result is explicitly supported.

**Key insight from research**: The existing split CSVs (`whisper-modeling/seen_child_splits/`) already contain per-clip manual video quality scores (`Video_Quality_*`) and visibility flags (`Child_of_interest_clear`, `#_adults`, `#_children`) — these enable an immediate MVP without any video processing.

## Technical Context

**Language/Version**: Python 3.11 (conda `child-vocalizations` env — same as MIL sweep)  
**Primary Dependencies**: `opencv-python` (YuNet face detector), `xgboost`, `scikit-learn`, `pandas`, `numpy`, `matplotlib`, `seaborn`, `scipy`; optional: `mediapipe` (fallback detector); existing `video/` env for ASD stretch  
**Storage**: CSV files for features and predictions; `.pkl` for trained models; JSON for metrics and configs; per-clip face detection JSON cache (`av_fusion/face_track_cache/`)  
**Testing**: Manual verification via split integrity report + sanity-check script; no formal test suite (consistent with rest of repo)  
**Target Platform**: Linux (SLURM cluster for video extraction; login node for training/evaluation)  
**Project Type**: Research pipeline (batch processing scripts + offline evaluation)  
**Performance Goals**: Visual feature extraction for ~2,200 clips at 2 fps sampling; target < 4 hours on single CPU node. Training/eval is near-instant (tabular, ~1500 rows).  
**Constraints**: Group-wise child-level split integrity is non-negotiable; no test leakage; fusion models must be interpretable (feature importance required).  
**Scale/Scope**: ~2,200 total clips; 109 children; 2 age bands; 4 model classes; 5+ evaluation strata.

## Implementation Strategy

### MVP (US1 + US2 + US3 + US4 only — manual annotations as visual features)

1. US2 first: `build_av_feature_table.py` with only metadata + manual annotations → master feature table exists immediately
2. US3: `train_av_fusion.py` → four models trained; val metrics confirm audio-only is reasonable baseline
3. US4: `evaluate_av_fusion.py` → test metrics + stratified results; thesis-ready output
4. **STOP and validate**: Compare audio-only vs. gated AV on visually eligible subset; confirm null or conditional result
5. **THEN** US1: add automatic visual feature extraction (face detection) and merge into master table; retrain

### Full Scope (US1–US6)

1. MVP above → US1 (automatic visual features) → retrain → compare manual-only vs. manual+auto
2. US5: error analysis → identify which failure modes AV addresses
3. US6 (stretch): ASD extraction → add to master table → retrain → compare

### User Story Dependencies

- **US1** (visual extraction): independent; can run in parallel with US2 MVP
- **US2** (feature table): depends on US1 being optional; can run with manual annotations only
- **US3** (training): depends on US2 output
- **US4** (evaluation): depends on US3 trained models
- **US5** (error analysis): depends on US4 predictions CSV
- **US6** (ASD): optional; parallel with US1; feeds into US2 if run

## Key Design Decisions Summary

| Decision | Choice | Reference |
|----------|--------|-----------|
| Visual feature fast-track | Manual BIDS annotations from split CSV (no video needed) | research.md §1 |
| Face detector | YuNet (OpenCV, no extra install) | research.md §2 |
| Face tracking | Simple IoU centroid tracker (~100 lines) | research.md §3 |
| Child candidate heuristic | Smallest median face bounding box | research.md §4 |
| Eligibility score formula | Weighted combination of 4 sub-scores | research.md §5 |
| Fusion model | XGBoost (NaN-safe) + LR (interpretable) | research.md §6 |
| Audio baseline source | BabAR `enroll_proba` (AUROC ~0.820) | research.md §7 |
| Output directory | `av_fusion/av_results/{run}/` | research.md §8 |
| ASD path | Reuse `video/run_asd.py` subprocess pattern | research.md §9 |
| Video path source | `BidsProcessed` column already in split CSV | research.md §10 |
