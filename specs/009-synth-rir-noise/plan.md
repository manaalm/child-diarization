# Implementation Plan: Synthetic Scene Acoustic Realism & Child Encoder Adaptation

**Branch**: `009-synth-rir-noise` | **Date**: 2026-04-26 | **Spec**: [spec.md](./spec.md)

## Summary

The cut-and-paste scene generator (`synth/scene_generator.py`) already imports
`convolve_rir` and `mix_at_snr` from `audio_utils.py` but never calls them — the
`_mix_scene_audio` method leaves `rir_id`, `noise_id`, and `mean_snr_db` as `None`.
This plan wires those helpers in (Stories 1–2 / FR-001–FR-009, FR-014) and
documents the stretch SSL pretraining path (Story 3 / FR-010–FR-013).

TinyVox ingestion into `build_segment_manifest.py` (FR-014) is **already implemented**
as of 2026-04-26 (`_scan_tinyvox()` + `--tinyvox-dir`/`--tinyvox-lang-filter` args).

The ratio-sweep SLURM job (12603925) is **currently running** on clean-mix scenes;
its results are the clean-mix baseline for SC-003.

## Implementation Phases

### Phase 1 — Wire RIR + Noise (P1, unblocked)

**T1. Fix `low_snr_stress.yaml`**: change `snr_db_min: -5` → `snr_db_min: 0`
(FR-007 / SC-002 require SNR ∈ [0, 5]).

**T2. Extend `SceneComposer.__init__`**: load RIR pool and noise pool at init time
from `mixing.rir_dir` and `mixing.noise_dir` config keys (or CLI overrides).
- Scan `rir_dir/*.wav` (and `*.flac`) recursively → `self._rir_pool: list[Path]`
- Scan `noise_dir/**/*.wav` → `self._noise_pool: list[Path]`
- Graceful fallback: if dir absent or empty, pool is `[]`; log a warning once
- Edge case: bad/corrupted files skipped at load time with per-file warning (FR-005)

**T3. Implement RIR + noise in `_mix_scene_audio`** (FR-001–FR-008):
After the existing speaker-track loop builds `mix`:
1. **RIR** (if `self._rir_pool` non-empty and `rng.random() < apply_rir_probability`):
   - Sample one RIR file from pool → `rir_path`; derive `rir_id = rir_path.stem`
   - Load RIR WAV with soundfile; resample to 16k if needed
   - Call `convolve_rir(mix, rir_wav)` → reverberant mix
   - Clamp output to float32 range
2. **Noise** (if `self._noise_pool` non-empty and `rng.random() < apply_noise_probability`):
   - Sample SNR: `snr_db = rng.uniform(snr_db_min, snr_db_max)`; clamp to range
   - Sample one noise file from pool → `noise_path`; derive `noise_id = noise_path.stem`
   - Load noise WAV; resample to 16k if needed
   - Call `mix_at_snr(mix, noise_wav, snr_db)` → noisy mix
3. Return `(mix, mean_snr_db, rir_id, noise_id)` — already consumed correctly by
   `generate_scene()` at lines 536–538 of `scene_generator.py`
4. Graceful fallback: if pool is empty, skip and leave `rir_id`/`noise_id` = `None`

**T4. Add `--rir-dir` and `--noise-dir` to `generate_scenes.py`** and inject into
the config dict before constructing `SceneComposer`. These override `mixing.rir_dir`
and `mixing.noise_dir` from the YAML, allowing cluster paths to be passed at runtime
without editing committed config files.

**T5. Update SLURM script** `run_scene_generation.sh` to forward `--rir-dir` and
`--noise-dir` args (pass-through from env vars or script arguments).

**T6. Update configs**: add `mixing.rir_dir: ""` and `mixing.noise_dir: ""` as
default-empty keys in all YAML configs (empty string → fall back to clean mix).

### Phase 2 — Re-run Manifest + Scenes (P1 prerequisite for P2)

**T7. Re-run `build_segment_manifest.py`** with `--tinyvox-dir data/tinyvox/` to
produce a manifest that includes both Providence and TinyVox child segments.
(FR-014 code is already done; this is the execution step.)

**T8. Re-generate 5000 scenes** via `run_scene_generation.sh default_14_18mo.yaml`
with `--rir-dir` and `--noise-dir` pointing to cluster data. If RIR/noise dirs are
unavailable at generation time, the graceful fallback (clean mix) still passes
FR-005; acoustic realism requires the data paths to be populated first.

**Note on data availability**: MUSAN and RIR datasets were not found in a quick
cluster search. Before running T8, the user must confirm their cluster paths or
arrange download (see research.md). Scene re-generation is blocked on this.

### Phase 3 — Ratio Sweep with Acoustic Scenes (P2)

**T9. Re-run `run_ratio_sweep.sh`** once acoustic scenes exist, targeting
`synth_results/augmentation_experiments/default_14_18mo_acoustic/`.

**T10. Run `evaluate_synthetic_augmentation.py`** and
`error_analysis_synthetic.py` on acoustic-mix results, then compare
`metrics_by_ratio.csv` against the clean-mix baseline from job 12603925.

### Phase 4 — SSL Pretraining (P3 Stretch)

**T11. Create `synth/scripts/pretrain_wavlm_child.py`**: accepts a WavLM-Base+
checkpoint + a file list of child speech WAVs (TinyVox + Providence), continues
masked-speech-unit prediction for a configurable number of steps, saves checkpoints,
logs training loss.

**T12. Create `synth/slurm/run_wavlm_pretrain.sh`**: 48h GPU job targeting
`ou_bcs_normal,pi_satra`, restartable via `--resume-from-checkpoint`.

**T13. Validate child-adapted checkpoint**: load into existing `BackboneExtractor`
in `mil/mil_model.py` (no code change), confirm MIL training runs without error and
AUPRC does not decrease vs. WavLM-Base+ baseline.
